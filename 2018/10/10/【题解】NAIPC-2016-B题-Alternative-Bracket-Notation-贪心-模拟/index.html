B. Alternative Bracket Notation

题目链接:https://nanti.jisuanke.com/t/32226

原题

Balanced closed bracket or parenthesis statements are ones where each opening bracket is matched with a closed bracket later in the string.

Notice how each closed parenthesis matches to the most recent unmatched open parenthesis.

Define an alternative bracket notation as follows: each bracket pair corresponds to a header in the form of “start,end:start,end:” where startstart and endend are indices of the new string itself! The index startstart is the index of the character immediately after the ‘::’, and endend is the index past the last header corresponding to the last bracket pair contained in this bracket pair. By taking a substring(start, end)(start,end) of the new notation, you get an alternative bracket sequence describing all of the pairs of brackets contained by the brackets corresponding to the “start,end:start,end:”! Since an empty pair of brackets has nothing inside, in their header, startstart and endend will be the same.

Each index takes up as many characters in the string as they do when they are base 1010 numbers. (For example, the index 4242 will take up 22 characters). The indices in the new string start from 00. All of the indices found in the alternative bracket notation string are absolute indices from the beginning of the new string.

Consider this parenthetical statement: (())(())

Here is it, in our new, alternate bracket notation: 4,8:8,8:4,8:8,8:

In this example, there are two sets of matching parenthesis, the outer one and the inner one. The outer one appears before the inner one, since the start bracket appears first. So, the header for the outer brackets will appear before the header for the inner bracket. The header 4,8:4,8: represents the outer bracket, while the header 8,8:8,8: represents the inner bracket. The substring from the 44th character to 77th character is 8,8:8,8:, which represents what is contained inside the outer bracket. Note that 5,11:11,11:5,11:11,11: could also be a legitimate alternate notation, but we want the shortest one, which is why 4,8:8,8:4,8:8,8: is the correct answer.

Input

Each input will consist of a single test case. Note that your program may be run multiple times on different inputs. The input will consist of a single line, containing a string ss, which consists only of open and closed parentheses. The string ss will be between 22 and 4,0004,000 characters long. There will be no spaces. The string ss is guaranteed to be balanced.

Output

Output the string ss in our new alternative bracket notation. If there’s more than one way to represent ss in the new notation, choose the shortest representation, which will be unique.

题意

用一串数字，按照题目的要求描述所给的括号信息。顺序依照每一对左括号出现的顺序，begin代表该括号的’(‘接下来一对的位置，end代表该括号的’)’接下来一对的位置。

题解

括号匹配关系用栈记录即可。

注意到位置会受到每个数字长度的影响，长度的改变会影响到所有后面信息的记录，牵一发而动全身。不妨假设最开始每个位置信息长度都为1，顺序扫描多次，修正每个位置信息，直到全部修正为止。

由于N最大为4000，易知最大的位置大小最多为5位数，扫描数P不超过5次。

复杂度O(PN^2)

当然，代码可以继续优化，在Get函数里，很多长度是重复算的，只需要记录一下就可以将复杂度优化为O(PN)
